# SyncMark iOS App — 设计方案

## 技术选型

| 项目       | 选择                          | 理由                                    |
|----------|-----------------------------|-----------------------------------------|
| UI 框架    | SwiftUI + iOS 17+           | 声明式、原生动画流畅、与后端推送集成好       |
| 网络层     | URLSession + async/await    | 原生够用，不引入三方依赖                   |
| WebSocket | URLSessionWebSocketTask     | 原生支持，配合 AsyncStream 使用           |
| 本地存储   | SwiftData                   | iOS 17 原生 ORM，替代 CoreData            |
| 图表       | Swift Charts                | 原生 K 线 / 指标图表                      |
| 推送       | APNs                        | 需求文档已定                              |
| 架构       | MVVM + Repository           | SwiftUI 天然 MVVM，Repository 隔离数据源  |

## 导航结构

```
TabBar (5 tabs)
├── 首页 (Dashboard)
├── 行情 (Market)
├── 链上 (On-Chain)
├── 日历 (Macro)
└── AI (Ask)

全局：
├── 通知中心 (右上角铃铛) — 大单 / 巨鲸 / 经济数据提醒
└── 设置 — 推送偏好、阈值配置、币种自选
```

```

│ ← 通知                          │
├──────────────────────────────────┤
│ 今天                             │
│ 💰 BTC 大单买入 $2.1M   14:32   │
│ 🐋 5000 ETH → Binance   14:20   │
│ 📅 非农就业 3小时后公布    14:00   │
│ 🟢 Paradigm 增持 ETH     13:45   │
│ ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈ │
│ 昨天                             │
│ ...                              │
└──────────────────────────────────┘
```

**推送类型（可在设置中按类型开关）：**

| 类型         | 触发条件                       | 优先级  |
|------------|------------------------------|--------|
| 大单警报     | value_usd > 用户设定阈值        | high   |
| 巨鲸转账     | severity = high / extreme     | high   |
| 聪明钱动向   | smart money buy/sell          | medium |
| 快讯         | importance = high             | medium |
| 经济数据提醒 | 重要事件公布前 30min            | high   |
| 经济数据结果 | 实际值 ≠ 预期值 (偏离 > 阈值)    | high   |

## 数据流设计

### 连接策略

```
App 启动
  ├── 建立 4 条 WebSocket 连接 (tickers / news / large-orders / whales)
  ├── REST 初始加载 (dashboard, news list, calendar)
  └── 注册 APNs token → 后端存储

App 进前台 → 恢复 WebSocket
App 进后台 → 断开 WebSocket，依赖 APNs 推送

WebSocket 断线 → 指数退避重连 (1s, 2s, 4s, 8s, max 30s)
                → 重连后 REST 补拉断线期间数据
```

### 离线 / 弱网

- SwiftData 本地缓存最近 500 条快讯、100 条巨鲸记录、7 天 K 线
- 无网络时展示缓存数据 + 顶部横幅提示 "网络连接中断"
- 恢复网络后自动增量同步

### 数据刷新频率

| 数据       | 方式                | 频率             |
|----------|-------------------|-----------------|
| Ticker   | WebSocket 推送     | 实时 (~500ms)    |
| 快讯       | WebSocket 推送     | 实时              |
| 大单/巨鲸  | WebSocket 推送     | 实时              |
| K 线      | REST 拉取          | 进入详情页时加载    |
| 经济日历   | REST 拉取          | 每次进入 Tab 刷新  |
| 仪表盘指标 | REST 拉取          | 60s 轮询          |
| AI 问答   | REST 请求/响应      | 用户触发           |

## 后端需要补充的工作

当前后端缺少 api-server 层。iOS 开发前需要先完成：

| 优先级 | 工作项                                            |
|-----|-------------------------------------------------|
| P0  | 实现 api-server (Go)：REST API 全部接口              |
| P0  | 实现 4 条 WebSocket 通道 (tickers/news/large-orders/whales) |
| P0  | APNs 推送集成 (设备注册 + 服务端推送)                   |
| P1  | K 线历史数据持久化 + 查询 API                          |
| P1  | 多所价差计算接口                                     |
| P2  | 链上概览聚合接口 (Gas / TVL / 活跃地址)                 |
| P2  | CME 降息概率数据源接入                                 |
